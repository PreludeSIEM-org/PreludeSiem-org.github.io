Here's another go at it:

Apparently there is a bug in SWIG similar to:

http://sourceforge.net/tracker/index.php?func=detail&aid=1388343&group_id=1645&atid=301645

Where init() isn't properly handled in java so, we had to expose init manually in c++/include/prelude.hxx (see bindings/java/prelude.hxx.patch)

once that was done, things compiled OK and bindings/java/testapp/test.java worked, as long as you call:

com.prelude.easy.PreludeEasy.prelude_init();

in your java code (Start() alone doesn't seem to cut it)

It's still a bit rough, but it appears to work from my dev box. I dunno if you're a bit more familiar with SWIG than I am and can send a patch up-stream to them.
