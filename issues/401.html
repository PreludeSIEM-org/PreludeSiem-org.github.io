Description

Hello everybody,

I recently installed the whole prelude suite on the Debian 6.0 Squeeze
machines using the provided packages (most recent packages of the 1.0.0
Prelude version). Everything works fine, but when I was browsing the
Prewikka interface, I discovered, that on the detailed view page of one
particular alert, the section ADITIONAL DATA contains somehow
encrypted/encoded information (please see the attached screenshot). In
my opinion, in this particular example fields 'Log received from' and
'Original Log' should be in readable form. Other fields are not
affected, for example 'Text' and 'Description' are readable.

I checked the database directly and table 'prelude_additionaldata'
contains these encrypted/encoded information as well, database tables
'prelude_classification', 'prelude_impact' and other on the other hand
contain unencrypted/unencoded data.

Is this the feature of the system? If it is, is it somehow possible to
display the information in clear text form (because this way it is
useless for me).

Or is it a bug in prelude? Isn't it encoding something it shouldn't or
forgets to decode after reception?

Or is it a bug in Prewikka interface, that forgets to decode after
reading from database?

Or is it my mistake, some misconfiguration?

Everything else seems normal and works properly. Does anyone else
encountered the same problem/behaviour as I did?

I was not sure about posting this here, but I was unable to post a message into the mailing list. I tried it many times from multiple different accounts (all properly subscribed), but the message did not appear in the mailing list and I have not received any errors back. I am receiving messages from other members in the list. There was no response on reporting the unability to post messages on mailboxes prelude-user-owner and postmaster@prelude-technologies.com, so I decided to share my problem here.

We have the same problem with Postgresql under Debian but not under CentOS. So it seems to be a Postgresql configuration issue on Debian.

#3 Updated by Jean-Charles ROGEZ almost 12 years ago

In PostGreSQL 9.x, bytea output now appears in hex format by default (see http://www.postgresql.org/docs/devel/static/release-9-0.html).

To work with Prelude, the following parameter must be specified in postgresql.conf.

bytea-output='escape'

We must update the install documentation.

#4 Updated by Antoine LUONG almost 12 years ago

The correct parameter is bytea_output.
