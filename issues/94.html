Description

When receiving logs via syslog-udp server, analyzer(0).node.name as well as the IP address are not set. Yet this information is available as it appears in additional data.

History

#1 Updated by Yoann VANDOORSELAERE about 19 years ago

We can not make the assumption that the source IP address where we read the UDP syslog message from is the same as the analyzer IP address. Maybe an option could be added to get the specific behavior you are looking for.

#2 Updated by Yoann VANDOORSELAERE almost 19 years ago

  • Status changed from New to Assigned

I am looking for use case with this specific behavior.

As stated above we can not make the assumption that the UDP sender is the original source for this message. Rather, LML is supposed to use the hostname within the SYSLOG header in order to deduce from what machine the log was generated.

This generally require that you run, for example, syslog-ng with:

keep_hostname(yes)
chain_hostname(no)

Do you have concrete use case for the behavior you are looking for ?
