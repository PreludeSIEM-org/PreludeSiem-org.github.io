#4 Updated by over 20 years ago

If you are on BSD (or anything that is not GNU in fact) you should use gmake to compile prelude (which invokes GNU make) instead of make (which invokes its BSD flavor). BSD make doesn't correctly understand makefiles created by autotools what causes all kinds of weird errors.

The official documentation should be updated to reflect this.

Now if you compile with gmake and you still have these errors that would mean that there is a problem... But I doubt ;-)
