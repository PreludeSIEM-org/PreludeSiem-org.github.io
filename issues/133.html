Description

Occasionally i snort instances hang and dont generate any alerts to prelude.
i have found the error below to be the cause on my system. (2.6.9-1.3_FC2smp).

"--- SIGRTMIN (Unknown signal 32) ---
sigreturn() = ? (mask now [])
rt_sigsuspend([]"

It seems that the prelude plugin in sending a SIGRTMIN signal back to snort for some reason. The only way i found i could get around this problem was to change the snort code in snort.c to include a signal handler for SIGRTMIN. This fixes the problem...however im not sure if this is a signal snort should be handling in the first place or my prelude configuration is generating this signal because there really is something wrong with prelude.

The SIGRTMIN signal is not used directly by Snort or Libprelude, but is probably triggered by your Linuxthread implementation (which since running on a recent kernel, might not use SIGUSR1 and SIGUSR2). Could you please answer the following question:

  • Glibc version ? (Exact RedHat package version if possible as well).
  • Output of: getconf GNU_LIBPTHREAD_VERSION
  • Is Prelude-Manager running on the same machine ? If yes, do you experience similar issue with it ? If no, is there another machine with the same kernel/glibc, running prelude-manager without problem ?
  • Does the issue still happen when replacing the pthread_sigmask() SIG_BLOCK argument with SIG_SETMASK in prelude-async.c ?
  • Does the issue still happen when fully removing the call to pthread_sigmask() in prelude-async.c ?
  • Does the issue still happen when removing the call to "signal(SIGUSR1, SigUsrHandler)" in snort.c ?
  • Is there anything specific that make the Snort process hang (for example, did you sent a SIGUSR1 signal to snort in order to get statistics before the process stalled) ?

Could you please also post the full, compressed strace output ?
