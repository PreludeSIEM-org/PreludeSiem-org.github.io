#1 Updated by Sebastien Tricaud over 17 years ago

It's pretty clear.

"No module named preludedb" means python can't find the libpreludedb pythond bindings.

I guess you haven't compile libpreludedb with python support, and to do it you need python devel headers. On debian, this package is called "python-dev".

Install it and run configure again in libpreludedb and it should work.

  • Status changed from Feedback to Closed

If you installed libpreludedb as an user, in a non root prefix, the Python bindings are not installed in the default Python module location. You then need to set the PYTHONPATH variable so that python know where to look up the libprelude/libpreludedb module.

This is not a bug, please raise this issue on the mailing list if this doesn't help.
