Prelude LML Rules¶

This project is aimed to gather and share LML Rules between PreludeOSS users.

If you have developped or modified LML rules this is the place to share them with the community.

How to contribute :
  • If you have a rule, create a feature ticket and put the rule in the ticket with little explanation
  • if you need a rule, create a feature ticket to ask for this rule

You can also contribute with log files examples (we will shortly publish an anonymizer script)

In case you don't want to subscribe to the RedMine project you can send rules in the user mailing list https://www.prelude-ids.org/lists/listinfo ( or even on the contact.prelude@c-s.fr email adress)

OSS projects can't survive (long) without contribution.

Thanks

The Prelude Team

Nota : As any contribution on PreludeOSS your contribution will be and stay GPL, you will be anoted as "author" of the rule but copyright will be for CSSI so we can include your rule in the pro edition.
